# OnceWorld Tools 実装計画詳細

## タスク一覧と担当区分

- 🤖 **Claude/Codeモード**: 自動実装可能

## Phase 1: 基盤構築（1週間）

### 1.1 Supabaseセットアップ

| # | タスク | 担当 | 所要時間 | 詳細 |
|---|--------|------|---------|------|
| 1.1.1 | Supabaseアカウント作成 | 👤 | 10分 | [supabase.com](https://supabase.com) でGitHubアカウントでサインアップ |
| 1.1.2 | 新プロジェクト作成 | 👤 | 5分 | プロジェクト名: onceworld-tools-db、リージョン: Tokyo |
| 1.1.3 | APIキー・URL取得 | 👤 | 5分 | Project Settings > API から `anon public` と `URL` をコピー |
| 1.1.4 | 環境変数ファイル作成 | 🤖 | 5分 | `.env.local` に `VITE_SUPABASE_URL` と `VITE_SUPABASE_ANON_KEY` を設定 |

### 1.2 データベース設計・作成

| # | タスク | 担当 | 所要時間 | 詳細 |
|---|--------|------|---------|------|
| 1.2.1 | SQLエディタでテーブル作成 | 🤖 | 30分 | Supabase Dashboard > SQL Editor で実行 |
| 1.2.2 | インデックス設定 | 🤖 | 15分 | 検索頻度の高いカラムにインデックス作成 |
| 1.2.3 | RLSポリシー設定 | 🤖 | 20分 | Row Level Securityで読み取りは公開、書き込みは認証必須に |
| 1.2.4 | テストデータ挿入 | 🤖 | 15分 | 各テーブルに1件ずつテストデータをINSERT |

### 1.3 TypeScript型定義

| # | タスク | 担当 | 所要時間 | 詳細 |
|---|--------|------|---------|------|
| 1.3.1 | Supabase CLIインストール | 🤖 | 10分 | `npm install -D supabase` |
| 1.3.2 | 型定義ファイル生成 | 🤖 | 10分 | `supabase gen types typescript` で自動生成 |
| 1.3.3 | 型定義の整理・配置 | 🤖 | 20分 | `src/types/supabase.ts` に配置 |

### 1.4 既存データ移行

| # | タスク | 担当 | 所要時間 | 詳細 |
|---|--------|------|---------|------|
| 1.4.1 | 既存JSONデータ確認 | 🤝 | 30分 | `docs/wiki/json/` のデータ構造を確認 |
| 1.4.2 | データ変換スクリプト作成 | 🤖 | 1時間 | JSON → Supabase INSERT用スクリプト |
| 1.4.3 | データ移行実行 | 👤 | 30分 | スクリプトを実行してデータを移行 |
| 1.4.4 | データ整合性確認 | 👤 | 30分 | 件数・内容が正しいか確認 |

## 実装スケジュール例

### 週1: Phase 1（基盤構築）
- **月曜**: Supabaseセットアップ（あなた: 30分）
- **火曜〜水曜**: データベース作成・型定義（Claude）
- **木曜**: データ移行（あなた: 1時間、Claude: 1時間）
- **金曜**: 整合性確認・Phase 1完了

### 週2: Phase 2（フロントエンド改修）
- **月曜〜水曜**: Supabase連携・リアルタイム機能（Claude）
- **木曜〜金曜**: 動作確認・調整（あなた: 2時間）

### 週3-4: Phase 3（管理画面作成）
- **週3**: 管理画面プロジェクト・認証（Claude）
- **週4**: CRUD画面・CSVインポート（Claude）
- **週末**: デプロイ（あなた: 1時間）

### 週5: Phase 4（オプション）
- 自動化機能の実装（Claude: 4時間、あなた: 30分）
