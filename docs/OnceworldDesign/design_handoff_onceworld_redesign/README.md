# Handoff: OnceWorld Tools リデザイン (A/Bテーマ切り替え)

## Overview

OnceWorld Tools (https://onceworld-tools.vercel.app / React + TypeScript + Vite) の UI リデザイン。
対象は当面 **ダメージ計算タブ** で、ここで確立したデザインシステムを他タブ (天空回廊 / ステータス / ペットバトル) に展開していく。

リデザインの柱は2つ:

1. **情報設計の整理** — 「① 敵を選ぶ → ② 自分のステータス → ③ 結果 (与ダメ / 被ダメ)」という操作の流れを、番号付きカードで画面構造として明示する。
2. **2テーマ切り替え** — ユーザーがヘッダーのスイッチで「ミニマル (Theme A / Clarity)」と「ファンタジー (Theme B / Grimoire)」を切り替えられる。選択は永続化する。

## About the Design Files

このバンドルの `reference/` 内のファイルは **HTMLで作られたデザインリファレンス** です。
意図した見た目と挙動を示すプロトタイプであって、そのまま本番に載せるコードではありません。

タスクは、これらのデザインを **対象コードベース (onceworld-tools: React + TypeScript + Vite) の既存の構成・パターンの中で再現すること** です。既存の計算ロジック・状態管理・ルーティングには手を入れず、見た目のレイヤーを置き換えてください。

## Fidelity

**High-fidelity (hifi)** です。色・タイポグラフィ・余白・角丸・状態はこのドキュメントと `tokens.css` の値をそのまま使ってください。
ただし以下は **モックの省略部分** なので、実装時は実アプリの仕様に従うこと:

- 計算結果の数値はスクリーンショットから採取した固定値 (一部推定値) — 実装では実際の計算結果を表示
- セレクトボックス・入力欄はダミー — 実装では既存のフォーム実装を流用
- 「装備 / アクセ / ペット / その他」サブタブの中身は未デザイン (既存UIを新トークンで再スタイル)

## Screens / Views

### 1. ヘッダー (全タブ共通)

- **Purpose**: タブ間ナビゲーション + ユーティリティ + テーマ切り替え
- **Layout**: `position: sticky; top: 0`、高さ 60px、横 padding 24px、`display: flex; gap: 24px`。
  左からロゴ → タブナビ → (右寄せ) ユーティリティリンク群 → テーマスイッチ。
- **Components**:
  - ロゴ: 12×12px の `accent` 色の正方形を 45° 回転させたマーク + 「OnceWorld Tools」(16px / 700 / 見出しフォント)。Theme B ではマークの角丸なし。
  - タブナビ: 14px、非アクティブ `muted`、アクティブは `ink` + 700 + 下線 2px `accent`。
  - ユーティリティ: 機能説明 / フィードバック / 更新履歴 / 装備確認 (12px / muted)。**1200px以下では非表示** (テーマスイッチを優先)。
  - テーマスイッチ: ピル型セグメント。各項目は 11px/700、選択中は `accent` 背景 + `accent-ink` 文字。項目内に 8px の回転正方形ドット。
- **背景**: `card`。下線は Theme A: 1px `line` / Theme B: 2px `accent`。

### 2. ダメージ計算タブ

コンテンツは `max-width: 1480px; margin: 0 auto`、padding `20px 24px 48px`。

## Interactions & Behavior

- **テーマ切り替え**: ルート要素の `data-theme="a" | "b"` を切り替える。CSS はすべて CSS カスタムプロパティ + `[data-theme]` スコープで分岐。選択は `localStorage` (キー例: `ow-theme`) に保存し、初期化時に復元。
- **トランジション**: テーマ切替時 `background / color / border-color 0.2〜0.3s`。レイアウトは動かさない (角丸・フォントは瞬時切替で OK)。
- **URLをコピー**: 既存のシェア機能をそのまま。フィードバックはトースト (画面上部中央、ink 背景ピル、1.8s で自動消滅、フェード+6px スライド)。
- **ホバー**: ナビ/ユーティリティ = 文字色が muted → ink。アクションリンク = 下線。ゴーストボタン = `accent-soft` 背景。
- **セグメント/サブタブ**: 即時切替。A のセグメント選択中は白背景 + `0 1px 2px rgba(31,29,24,0.08)` の影。
- **Reduced motion**: テーマトランジションは `prefers-reduced-motion` で無効化推奨 (モック未実装)。

## State Management

- `theme: 'a' | 'b'` — グローバル (Context など既存のパターンに従う)。localStorage と同期。
- 文言テーブル `AB_LABELS` (reference/ab-app.jsx 参照): テーマで見出し・補助文言が変わる (「与ダメージ」⇄「討伐の見立て」等)。**文言切り替えが過剰なら Theme B でも A の文言に統一して良い** (発注者と要相談)。
- それ以外の状態 (選択モンスター、Sim 設定など) は既存実装のまま。

## Design Rules (根拠と判断基準)

1. **流れを構造で示す** — 入力と結果が混在して見えるのが現行の課題。①②③のステップ番号と「入力は左、結果は右」の原則を崩さない。
2. **結論ファースト** — ユーザーの最終関心は「何回で倒せるか / 何回耐えられるか」。撃破回数バッジと「受けられる回数」カードは常に画面内で最も強いコントラストを持つ。
3. **数字が主役** — 桁の大きい数値が情報の本体。tabular-nums で桁を揃え、ダメージレンジは行内最大 (16px/700)。装飾が数字の可読性に勝ってはいけない。
4. **属性色はアイコンとチップに限定** — 5属性色を面で使うと騒がしくなる。背景や枠への使用は禁止。UI のアクセントは各テーマ1色 (青 / 金) のみ。
5. **テーマは「皮」** — A と B で DOM 構造は完全に同一。差分は CSS 変数と `[data-theme]` スコープのルール (+文言テーブル) だけ。これを実装の受け入れ条件とする。
6. **警告色は不足情報のみ** — `--warn` は「あと X 足りない」系の情報専用。
7. **絵文字・手描きSVGイラストは使わない** — 図形は正方形/円/45°回転 (ダイヤ) まで。
