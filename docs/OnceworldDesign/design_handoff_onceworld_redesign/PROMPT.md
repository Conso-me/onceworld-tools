# Claude Code への依頼プロンプト

以下をそのまま Claude Code に貼り付けて使えます (リポジトリのルートでこのフォルダを `design_handoff_onceworld_redesign/` として置いた前提)。

---

OnceWorld Tools (このリポジトリ: React + TypeScript + Vite) の UI をリデザインします。
`design_handoff_onceworld_redesign/README.md` を最初に読んでください。デザインの正は
`design_handoff_onceworld_redesign/reference/AB Prototype.html`(ブラウザで開いて確認可能)と、
その実装 `reference/ab-app.jsx` です。トークンは `tokens.css` にまとまっています。

## ゴール

1. **テーマ基盤**: `data-theme="a" | "b"` + CSS カスタムプロパティによる2テーマシステムを導入する。
   - Theme A「ミニマル」(白基調・青アクセント・角丸) / Theme B「ファンタジー」(羊皮紙・金・明朝・額縁)
   - ヘッダー右にテーマスイッチ (ピル型セグメント) を置き、選択を localStorage に永続化
   - DOM 構造は両テーマで完全に同一。差分は CSS 変数・`[data-theme]` スコープの CSS・文言テーブルのみ
2. **ダメージ計算タブのレイアウト再構成**:
   - 「① 敵を選ぶ(全幅カード) → ② 自分のステータス(左 330px) → ③ 与ダメージ(中央可変) + 被ダメージ(右 350px)」のステップ構造
   - 各カードのヘッダにステップ番号バッジ
   - レスポンシブ: 1100px で2カラム、760px で1カラム + 下部固定サマリー (受けられる回数 / 被ダメ)
3. **既存ロジックは不変**: 計算・URL シェア・プリセット等の挙動は一切変えない。見た目のレイヤーだけ置き換える。

## 進め方

- まず既存のスタイリング手法 (CSS Modules / styled-components / Tailwind / 素のCSS など) を調べ、それに合わせて tokens.css の内容を移植する
- 次にテーマ Context + スイッチ + 永続化を実装
- その後ダメージ計算タブを README の「Screens / Views」仕様どおりに再構成
- 文言切り替え (AB_LABELS: 「与ダメージ」⇄「討伐の見立て」等) は最後。工数が嵩むならまず A の文言で統一し、TODO として残す

## 受け入れ条件

- [ ] テーマスイッチで全 UI の色・フォント・角丸が即時切り替わり、リロード後も維持される
- [ ] 両テーマで DOM が同一 (テーマ切替で再マウントされない)
- [ ] 数値がすべて tabular-nums で桁揃えされている
- [ ] 撃破回数バッジは「1回」のときだけ強調表示
- [ ] 「受けられる回数」が画面内で最も目立つ要素になっている
- [ ] 760px 以下で1カラム + 下部固定サマリーになり、横スクロールが発生しない
- [ ] 属性色 (火/水/木/光/闇) の使用はアイコンとチップのみ。背景・枠には使わない
- [ ] 既存の計算結果・シェアURL の挙動が回帰していない

## 注意

- reference/ の HTML/JSX はデザインリファレンスであり、コピペ移植しない (特に CSS-in-JS 文字列)。コードベースの流儀で書き直すこと
- モックの数値はダミー。実装では実データを流す
- 絵文字・手描き SVG イラストは使わない (図形は正方形/円/45°回転まで)
- 他タブ (天空回廊 / ステータス / ペットバトル) は今回のスコープ外だが、共通コンポーネント (カード・セグメント・チップ等) は再利用可能な形で切り出しておく
